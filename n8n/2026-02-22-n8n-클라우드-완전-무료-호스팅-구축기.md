### Context
n8n을 로컬이 아닌 클라우드에서 24시간 실행하고자 했습니다. 그러나 유료 서비스는 비용 부담이 있었기에 Google Cloud Platform(GCP)의 Always Free 티어를 활용하여 무료 인프라를 구축하는 방안을 탐색했습니다.

### Core
* **GCP e2-micro:** 구글 클라우드 프리트라이얼 인스턴스인 e2-micro를 활용하여 기본 서버 자원을 확보했습니다.
* **Docker:** `docker-compose`를 사용하여 n8n 환경을 컨테이너로 만들어 설치 및 관리 효율성을 극대화했습니다.
  ```yaml
  version: '3'
  services:
    n8n:
      image: n8nio/n8n
      ports:
        - "5678:5678"
      environment:
        - DB_TYPE=sqlite
        - DB_SQLITE_DATABASE=/home/node/.n8n/database.sqlite
  ```
* **Ngrok:** 고정 IP나 복잡한 방화벽 및 도메인 설정 없이 Ngrok 터널링을 통해 외부에서의 웹훅(Webhook) 경로 노출을 효과적으로 관리했습니다.
  ```bash
  ngrok http 5678
  ```

### Insight
유료 클라우드 서비스가 비싼 이유를 체감하게 되었습니다. 무료로 설정할 때 발생하는 여러 설정 이슈들은 설정 과정이 매우 까다롭고 시간이 소요되는 일이 많았습니다. 특히, 도메인 주소 설정 시 DuckDNS와 Let's Encrypt를 활용하려 했으나 CAA 레코드 오류가 발생하여 3시간 넘게 문제를 해결한 후 Ngrok을 사용하여 대체했습니다. 이러한 노력 끝에 100% 무료 조합을 완성할 수 있었습니다.
