### Context
n8n을 이용하여 디스코드(Discord)에서 특정 메시지를 수신했을 때 자동으로 워크플로우가 시작되도록 설정했습니다. 이 워크플로우는 메시지를 트리거로 하여 에이전트를 실행하고, TIL(Today I Learned) 형식의 문서를 생성하여 GitHub에 자동으로 푸시하고, 해당 내용을 README.md에 인덱스로 추가한 후 완료 메시지를 발신하는 절차를 포함합니다.

### Core
- **Trigger Node**: Discord Webhook을 통해 메시지 수신을 트리거로 설정.
- **Agent Execution**: 메시지 수신 시 자동으로 에이전트를 실행하여 내용을 TIL 형식으로 변환.
- **GitHub Push**: 변환된 내용을 GitHub 레포지토리에 자동으로 푸시.
- **README Update**: 푸시 완료 시 README.md에 인덱스를 추가.
- **Completion Message**: 모든 과정이 완료되면 최종 확인 메시지를 디스코드 채널에 발신.

```json
{
  "nodes": [
    {"name": "Discord Trigger", "type": "webhook"},
    {"name": "Agent Execution", "type": "executeFunction"},
    {"name": "GitHub Push", "type": "github"},
    {"name": "README Update", "type": "fileAppend"},
    {"name": "Completion Message", "type": "sendMessage"}
  ]
}
```

### Insight
n8n으로 워크플로우를 처음 만들어본 경험에서, 중요한 점은 각 노드의 입력(input)과 출력(output)을 명확히 이해하는 것입니다. 이러한 점만 알면 n8n을 활용한 자동화 설계가 매우 수월해집니다. 가이드를 제공해 준 후츠릿에게 감사합니다.
