### Context
n8n을 무료로 호스팅하고자 하는 개발자들은 종종 서버 비용과 설정의 복잡성 때문에 고민합니다. 특히 GCP의 Always Free 티어를 활용하여 비용을 최소화하려는 경우가 많습니다. 이때 Docker와 Ngrok를 조합하여 효율적인 설정이 가능합니다.

### Core
1. **GCP 설정**: GCP의 Always Free tier를 사용하여 VM 인스턴스를 생성합니다.
   * f1-micro 인스턴스를 선택하여 비용을 절감합니다.

2. **Docker 설치**: 인스턴스에 Docker를 설치합니다.
   ```bash
   sudo apt update
   sudo apt install docker.io
   sudo systemctl start docker
   sudo systemctl enable docker
   ```

3. **n8n Docker 설정**: n8n을 Docker 컨테이너로 실행합니다.
   ```bash
   docker run -it --rm \
     --name n8n \
     -p 5678:5678 \
     -e N8N_BASIC_AUTH_ACTIVE=true \
     -e N8N_BASIC_AUTH_USER=<USER> \
     -e N8N_BASIC_AUTH_PASSWORD=<PASSWORD> \
     n8nio/n8n
   ```

4. **Ngrok 설정**: 외부에서 접근할 수 있도록 Ngrok를 통해 터널링합니다.
   ```bash
   ./ngrok http 5678
   ```

### Insight
이 방법은 GCP의 무료 티어를 활용하여 지속적인 비용 없이 n8n 서버를 호스팅할 수 있도록 합니다. Docker를 사용하여 환경 설정을 자동화하고 Ngrok를 통해 안전하게 외부에 노출할 수 있어 테스트 또는 소규모 프로젝트에 적합합니다. 주의할 점은 Ngrok의 무료 플랜이 대역폭과 연결 제한이 있으므로 필요에 따라 유료 플랜을 고려해야 합니다.
