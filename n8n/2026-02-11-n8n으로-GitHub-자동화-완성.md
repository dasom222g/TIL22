### Context
n8n을 사용하여 GitHub과의 자동화를 구현했습니다. 여러 애플리케이션과 GitHub을 연동하여 자동화 워크플로우를 구축할 수 있는 가능성을 탐색하였습니다. 주된 목표는 소스 관리 프로세스를 효율화하는 것이었습니다.

### Core
n8n에서 GitHub과 연동하려면 다음과 같은 설정이 필요합니다:

* **GitHub Credentials 설정**: GitHub Developer settings에서 새로운 Personal Access Token을 생성하고, n8n에 Credential로 등록합니다.
* **n8n 워크플로우 작성**: 트리거 노드로 GitHub Webhook을 추가하고, 다양한 액션 노드(Post Issue, Get Repository Info 등)와 연결합니다.

```json
{
  "nodes": [
    {
      "type": "n8n-nodes-base.githubTrigger",
      "parameters": {
        "events": [
          "issues"
        ]
      },
      "name": "Github Trigger",
      "typeVersion": 1,
      "position": [250, 300]
    },
    {
      "type": "n8n-nodes-base.github",
      "parameters": {
        "operation": "create",
        "resource": "issueComment",
        "owner": "repo-owner",
        "repo": "repo-name",
        "issue_number": "={{$json[\"action\"]}}"
      },
      "name": "Create Comment",
      "typeVersion": 1,
      "position": [400, 300]
    }
  ]
}
```

### Insight
n8n으로 수행한 GitHub 자동화는 코드 리뷰, 이슈 관리 등을 자동화하여 개발 프로세스를 크게 개선했습니다. 워크플로우 설정 후 다양한 GitHub 작업을 자동으로 실행할 수 있었고, 이를 통해 팀의 생산성을 높일 수 있었습니다.
