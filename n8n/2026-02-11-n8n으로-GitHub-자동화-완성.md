### Context
GitHub 작업을 자동화하기 위해 n8n을 사용해 워크플로우를 구성했습니다. 이 자동화는 코딩 작업과 일상적인 GitHub 관리를 효율화하려는 목적으로 진행되었습니다.

### Core
n8n을 활용하여 GitHub 리포지토리에서 새로운 이슈가 생성될 때마다 Slack으로 알림을 보내는 워크플로우를 설정하였습니다. 주요 단계는 다음과 같습니다:

* **GitHub Trigger 노드**: 새로운 이슈 발행 시 트리거
* **HTTP Request 노드**: Slack의 웹훅 URL로 데이터 전송

```json
{
  "nodes": [
    {
      "name": "GitHub Trigger",
      "type": "n8n-nodes-base.github",
      "parameters": {
        "event": "issues",
        "owner": "my-github-username",
        "repo": "my-repo-name"
      }
    },
    {
      "name": "Slack Notification",
      "type": "n8n-nodes-base.httpRequest",
      "parameters": {
        "url": "https://hooks.slack.com/services/...",
        "method": "POST",
        "body": "{ \"text\": \"New issue created in GitHub!\" }"
      }
    }
  ]
}
```

### Insight
이 자동화 워크플로우를 통해 개발팀의 반응 시간이 크게 단축되었습니다. n8n의 시각적 UI를 통해 복잡한 프로그래밍 없이도 효율적인 작업이 가능했습니다. 이는 업무 생산성 향상에 큰 기여를 했습니다.
