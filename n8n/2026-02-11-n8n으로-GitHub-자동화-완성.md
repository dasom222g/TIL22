### Context
GitHub의 다양한 작업을 자동화하기 위해 n8n의 워크플로우 설계 기능을 활용할 수 있습니다. 이번에는 n8n을 사용하여 GitHub과의 통합을 통해 반복 작업을 자동화하는 방법을 학습했습니다.

### Core
n8n에서 GitHub 관련 작업을 자동화하기 위해 다음과 같은 워크플로우를 설정할 수 있습니다:

```yaml
to: github
from: n8n
trigger: GitHub 이벤트 (예: Issue 생성)
actions:
  - title: '신규 Issue 알림'
    nodes:
      - type: 'Webhook'
        config:
          url: 'https://api.n8n.io/webhook-url'
      - type: 'GitHub'
        config:
          update: true
          message: '새로운 이슈가 등록되었습니다: {{issue_title}}'
```

### Insight
이 워크플로우를 통해 특정 GitHub 이벤트가 발생할 때 자동으로 알림을 받거나, 특정 작업을 수행하도록 설정할 수 있습니다. 이로써 수작업으로 처리해야 하는 반복적인 작업을 줄이고 효율성을 높일 수 있습니다.
