### Context
n8n을 사용하여 디스코드에서 메시지를 수신하면 자동으로 GitHub에 TIL 형식의 문서를 업데이트하는 워크플로우를 구축했습니다. 이 자동화 과정은 개발 일지를 체계적으로 관리하고 팀과 공유할 수 있는 효율적인 방법입니다.

### Core
워크플로우는 다음 단계로 구성되어 있습니다:

* **트리거**: 디스코드에서 특정 메시지를 수신할 때 이벤트가 발생합니다.
* **Agent 실행**: 수신된 메시지를 TIL 형식으로 변환합니다.
* **GitHub에 push**: 변환된 내용을 GitHub 저장소에 push합니다.
* **README.md 업데이트**: 새로운 인덱스를 README.md 파일에 추가합니다.
* **완료 메시지 발신**: 프로세스가 완료되었음을 알리는 메시지를 발신합니다.

n8n에서 트리거와 각 노드의 input/output을 정확히 설정하면 전체 워크플로우를 쉽게 관리할 수 있습니다.

### Insight
n8n을 처음 사용해 보았는데, 두 가지 원칙만 이해하면 워크플로우 작성이 간단하다는 것을 확인했습니다. 각 노드가 어떤 input을 받고 어떤 output을 내보내는지를 이해하면 됩니다. 특히, Discord와 GitHub 사이에서 데이터를 효과적으로 주고받는 자동화를 구현하는 데 n8n이 무척 유용했습니다. 이러한 기능을 빠르게 습득할 수 있도록 도와준 후츠릿에게 감사드립니다.
