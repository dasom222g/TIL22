### Context
n8n을 사용하여 GitHub 저장소의 작업을 자동화하고자 했습니다. 반복적인 작업을 줄이고 자동 통지를 설정하여 개발 워크플로우를 효율화하려는 것이 주된 목표였습니다.

### Core
n8n을 통해 GitHub 자동화를 설정했습니다. 사용한 주요 노드는 다음과 같습니다:

* **Webhook 트리거**: GitHub에서 특정 이벤트 발생 시 n8n 워크플로우가 시작되도록 설정.
* **HTTP Request 노드**: GitHub API를 호출하여 이슈 및 PR 상태를 자동으로 갱신.
* **Slack 노드**: 업데이트된 정보에 대해 팀에게 자동으로 알림.

```json
{
  "nodes": [
    {
      "name": "Webhook",
      "parameters": {"httpMethod": "POST"},
      "type": "n8n-nodes-base.webhook",
      "typeVersion": 1
    },
    {
      "name": "HTTP Request",
      "parameters": {
        "url": "https://api.github.com/repos/{owner}/{repo}/issues",
        "method": "GET",
        "json": true
      },
      "type": "n8n-nodes-base.httpRequest",
      "typeVersion": 2
    },
    {
      "name": "Slack",
      "parameters": {
        "text": "New update on GitHub issue: {{$json[\"issue_title\"]}}",
        "channel": "#development"
      },
      "type": "n8n-nodes-base.slack",
      "typeVersion": 1
    }
  ]
}
```

### Insight
n8n을 활용하여 GitHub의 반복 작업을 대폭 줄이고, 특정 이벤트 발생 시 실시간으로 팀에게 통지할 수 있는 자동화 워크플로우를 구축했습니다. 자동화가 성공적으로 이루어지면, 작업 시간을 절약하고 개발 프로세스의 효율성이 높아집니다.
