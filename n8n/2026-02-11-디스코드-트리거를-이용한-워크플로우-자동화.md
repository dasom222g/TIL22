### Context
n8n을 활용하여 Discord에서 메시지를 수신하면 자동으로 GitHub에 TIL 형식의 문서를 생성하고 push하는 워크플로우를 만들었습니다. 이 과정에서는 메시지를 수신하여 트리거를 시작하고, Agent가 TIL 형식의 콘텐츠를 생성한 후 GitHub에 push합니다. 마지막으로 README.md 파일에 인덱스를 추가하고 완료 메시지를 발신합니다.

### Core
```json
[
  {
    "name": "Discord Trigger",
    "type": "trigger",
    "input": "Discord 메시지",
    "output": "Agent 기능 트리거"
  },
  {
    "name": "Agent",
    "type": "process",
    "input": "Discord 메시지",
    "output": "TIL 형식 변환된 파일"
  },
  {
    "name": "GitHub Push",
    "type": "action",
    "input": "변환된 파일",
    "output": "GitHub 리포지토리에 파일 저장"
  },
  {
    "name": "README.md 업데이트",
    "type": "action",
    "input": "작업 완료 인덱스",
    "output": "업데이트된 README.md"
  },
  {
    "name": "완료 메시지 발신",
    "type": "action",
    "input": "프로세스 완료",
    "output": "Discord로 완료 메시지"
  }
]
```

### Insight
n8n으로 처음 워크플로우를 구성했는데, 각 노드의 input과 output만 명확히 이해하면 설정이 어렵지 않습니다. 메시지를 수신하고 이를 변환해 저장하는 과정은 각각 별도의 노드로 구성되어 있으며, 이를 연결하는 방식으로 전체 워크플로우가 형성됩니다. 복잡해 보일 수 있지만, 후처리 노드를 통해 워크플로우를 매끄럽게 이어갈 수 있어 많은 도움이 되었습니다.
